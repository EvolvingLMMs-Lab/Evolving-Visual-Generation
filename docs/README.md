# Documentation Map

This folder turns the paper into a GitHub-friendly roadmap.

- [`taxonomy.md`](taxonomy.md): capability levels from Atomic Generation to World-Modeling Generation.
- [`technical_drivers.md`](technical_drivers.md): modeling paradigms, architectures, and system design.
- [`training_data_alignment.md`](training_data_alignment.md): data, training, post-training, reward modeling, and acceleration.
- [`applications.md`](applications.md): application regimes and why they require more than perceptual quality.
- [`stress_tests.md`](stress_tests.md): qualitative probes for spatial, physical, textual, and long-horizon failures.
- [`frontiers.md`](frontiers.md): positions and open research directions.
- [`reading_list.md`](reading_list.md): a curated entry point into the cited literature.

The root [`README.md`](../README.md) is the public landing page. These notes are intentionally concise so that the repo can grow into an awesome list, project page, or paper companion without duplicating the full manuscript.
